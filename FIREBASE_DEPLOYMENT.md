# Firebase Hosting deployment

The site is built into `dist/` and deployed to the live Firebase Hosting channel
after every push to `main`.

## One-time setup

1. Create or select a project in the [Firebase console](https://console.firebase.google.com/), then enable Hosting.
2. Initialize a Git repository, push this project to GitHub, and make `main` the default branch if that has not already been done.
3. From this project directory, run `npx firebase-tools init hosting:github` and follow the prompts. Select the existing `dist` directory as the public directory and do not overwrite `firebase.json`.
4. The CLI creates a service-account secret and may give it a project-specific name such as `FIREBASE_SERVICE_ACCOUNT_MY_PROJECT`. In `.github/workflows/deploy-firebase-hosting.yml`, replace `secrets.FIREBASE_SERVICE_ACCOUNT` with that generated secret name. Delete any extra workflow files generated by the CLI so each push deploys only once.
5. In the GitHub repository, open **Settings > Secrets and variables > Actions > Variables** and create a repository variable named `FIREBASE_PROJECT_ID` whose value is the Firebase project ID.
6. Push a commit to `main`, then verify the **Deploy to Firebase Hosting** run on the repository's **Actions** tab.

The Firebase CLI setup command creates a narrowly scoped deployment service account and stores its JSON key as an encrypted GitHub secret. Never commit that JSON key to this repository.

## Manual deployment

```sh
npm ci
npm test
npm run build
npx firebase-tools deploy --only hosting --project YOUR_FIREBASE_PROJECT_ID
```
